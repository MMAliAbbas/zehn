# LogicIgniter Solution Portfolio Plan

## Active Portfolio Model

LogicIgniter’s in-house SaaS portfolio is 51 applications organized into 10 canonical suites. All 51 apps launch together; no suite or app is independently launchable without Ali approval.

This file is current-state planning context. Older Ignite package names are historical and must not be used as active portfolio identity.

## Operating Rules

- CEO owns company-level synthesis and launch go/no-go recommendation.
- CPO owns product coherence, acceptance criteria, and suite readiness.
- CTO owns technical architecture, integration quality, and engineering risk.
- Specialist agents execute by discipline, not by one persistent agent per app.
- Durable portfolio corrections should be written to Yaad scope `organization:logicigniter`.
